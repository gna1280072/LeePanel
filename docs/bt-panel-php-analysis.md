# 宝塔面板 PHP 目录结构分析

## 宝塔的 PHP 安装方式

宝塔面板的 PHP 是**自己从源码编译的**，不是用 `apt` 装的。

## 目录结构

宝塔把每个 PHP 版本的所有东西都放在 `/www/server/php/XX/` 下：

```
/www/server/php/7.4/
├── bin/          # php, phpize, php-config 等二进制
├── sbin/         # php-fpm
├── etc/          # php.ini, php-fpm.conf, pool.d/
├── lib/          # 扩展 .so 文件
├── var/          # 日志、pid、socket
└── include/      # 头文件
```

通过 `./configure --prefix=/www/server/php/7.4` 指定安装路径，编译时把所有路径都指到这个目录下，实现自包含。

## 为什么宝塔要这么做

| 原因 | 说明 |
|------|------|
| 多版本共存 | apt 装多个 PHP 版本容易冲突，自己编译放不同目录互不干扰 |
| 统一路径 | 不管 CentOS 还是 Ubuntu，路径都一样，面板逻辑不用区分系统 |
| 自定义扩展 | 编译时可以按需加 `--with-xxx` 参数，灵活控制功能 |
| 卸载干净 | 删掉整个目录就卸载了，不留残余 |

## 代价

- 编译耗时长（安装 PHP 要等好几分钟）
- 不享受系统包管理器的安全更新（需要手动在面板里升级）
- 依赖库还是要从系统装（libxml2-dev、libssl-dev 等开发包）

## 总结

宝塔本质上是**源码编译 + 自定义路径**的方案，用编译时的 `--prefix` 参数把所有东西收拢到一个目录下，实现了"目录即安装"的效果。
