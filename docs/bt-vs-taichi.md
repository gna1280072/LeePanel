# 宝塔面板 vs TaiChi Panel 对比分析

## 架构本质

| 维度 | 宝塔面板 | TaiChi Panel |
|------|---------|-------------|
| **运行位置** | 安装在服务器端（B/S架构，浏览器访问 Web UI） | 桌面客户端（Tauri，本地运行，SSH 连接远程服务器） |
| **依赖** | 服务器上需要安装宝塔 Agent，占用资源 | 服务器零依赖，只需 SSH 即可管理 |
| **访问方式** | 通过 IP:8888 浏览器访问，需开放端口 | 本地桌面 App，无需在服务器开放额外端口 |
| **安全性** | Web 端口暴露，存在被扫描/爆破风险 | 无 Web 端口暴露，纯 SSH 通道 |

## 功能对比

| 功能模块 | 宝塔面板 | TaiChi Panel |
|---------|---------|-------------|
| **Dashboard** | 系统概览、负载、内存、磁盘 | ✅ Dashboard（MonitorPanel） |
| **终端** | Web Shell（浏览器内终端） | ✅ 内嵌 xterm.js 终端 |
| **文件管理** | 完整文件管理器（编辑、权限、压缩） | ✅ FileBrowser（拖拽上传、文件夹上传、编辑器） |
| **网站管理** | 一键建站、SSL、反向代理、伪静态 | ✅ Sites 面板（建站、SSL、反向代理） |
| **数据库** | MySQL/MariaDB/PostgreSQL 完整管理（phpMyAdmin） | ✅ Database 面板（MySQL/MariaDB） |
| **Redis** | 内置管理界面 | ✅ Redis 面板 |
| **Docker** | Docker 管理器（容器、镜像、compose） | ✅ Docker 面板（容器管理） |
| **软件仓库** | 一键安装各类软件（LNMP、Tomcat、Node等） | ✅ Software Repository（LNMP、Docker、Redis、Node等） |
| **防火墙** | 端口管理、IP黑白名单 | ✅ Firewall 面板（端口管理） |
| **计划任务** | ✅ 定时备份、Shell 脚本、日志切割 | ❌ 暂未实现 |
| **日志管理** | 系统日志、网站日志、面板日志 | ✅ Site Logs 面板 |
| **BBR 加速** | ❌ 需手动配置 | ✅ 一键开启 BBR |
| **SSH 密钥管理** | 部分支持 | ✅ 密钥生成、部署、密码/密钥切换 |
| **多服务器** | 需安装"宝塔云监控"插件 | ✅ 多连接管理（Sidebar 侧边栏） |
| **国际化** | 仅中文 | ✅ 中英文双语 |

## 核心差异

### 宝塔的优势

- 功能更完整（计划任务、备份、邮件通知、Webhook）
- 生态更成熟（插件市场、大量用户社区）
- 服务器端缓存/优化（面板本身可做本地缓存）
- PHP 管理更细（多版本切换、扩展安装、php.ini 可视化编辑）
- 支持 Windows Server（IIS 管理）

### TaiChi Panel 的优势

- **零服务器侵入**：不需要在服务器安装任何 Agent，纯 SSH 驱动
- **无端口暴露**：不需要开放 8888 等 Web 端口，攻击面小
- **离线可用**：客户端本地运行，不依赖面板服务器在线
- **多服务器管理更便捷**：侧边栏切换，无需"云监控"中间层
- **BBR 一键开启**：宝塔需要手动 SSH 配置
- **轻量**：Tauri 打包体积小，资源占用低（相比 Java/Node 的 Web 面板）
- **跨平台客户端**：Windows/Mac/Linux 桌面原生体验

## 总结

宝塔是**服务器端的全能管家**，功能大而全，但需要在服务器上安装 Agent 并暴露 Web 端口；TaiChi Panel 是**远程管理工具**，通过 SSH 通道实现零侵入管理，功能聚焦于常用场景，安全性更好，但功能覆盖度相对较窄（如缺少计划任务、备份管理）。两者定位不同，宝塔适合重度服务器运维，TaiChi Panel 适合轻量、安全、快速的日常管理。
