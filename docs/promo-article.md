# TaiChi Panel：下一代桌面级服务器管理工具

## 告别 Web 面板的安全隐患，用 SSH 重新定义服务器管理

---

### 你还在用宝塔面板吗？

2024 年，宝塔面板因安全漏洞导致大量服务器被入侵的事件仍在许多运维人员心中留有阴影。Web 面板的本质矛盾从未改变：**为了管理服务器，你必须在服务器上暴露一个 Web 端口**——这等于给黑客留了一扇永远敞开的门。

**TaiChi Panel** 从根本上解决了这个问题。

---

### 什么是 TaiChi Panel？

TaiChi Panel 是一款**桌面原生**的服务器管理工具，基于 Tauri 框架开发，通过 SSH 协议远程管理你的 Linux 服务器。

**不需要在服务器上安装任何东西。** 不需要开放额外端口。不需要 Agent。

你的服务器只需要 SSH——这个你本来就在用的、最安全的远程管理协议。

---

### 核心优势

#### 🔒 零侵入，零暴露

传统 Web 面板需要在服务器上安装 Agent、开放 Web 端口（如 8888、9090），这些端口常年暴露在公网，是黑客扫描和攻击的首要目标。

TaiChi Panel **不需要在服务器上安装任何软件**，也不需要开放任何额外端口。所有操作通过 SSH 加密通道完成，攻击面为零。

#### ️ 桌面原生体验

基于 Tauri 构建，TaiChi Panel 是真正的桌面应用——不是套壳浏览器。

- **Windows / macOS / Linux** 全平台支持
- 原生窗口、系统托盘、快捷键
- 离线可用，不依赖面板服务器在线
- 安装包体积小巧（约 10MB），启动秒开

#### ⚡ 全功能管理面板

虽然架构极简，但功能毫不妥协：

| 功能 | 说明 |

| 📊 **Dashboard** | 系统概览、CPU/内存/磁盘实时监控 |

#### 🌐 国际化

原生支持中文和英文，面向全球用户。

### 技术栈

- **前端**：React + TypeScript + Vite
- **后端**：Rust（Tauri 2.x）
- **SSH**：russh（纯 Rust SSH 实现）
- **终端**：xterm.js
- **国际化**：i18next

---

### 谁适合使用 TaiChi Panel？

- **独立开发者**：管理 1-10 台 VPS，不想装宝塔
- **运维工程师**：需要快速查看服务器状态、管理文件、操作数据库
- **安全敏感用户**：不想在服务器上暴露 Web 端口
- **多服务器管理者**：频繁切换不同服务器，需要统一管理界面
- **宝塔/1Panel 用户**：想要更轻量、更安全的替代方案

---

### 谁不适合？

- 需要**计划任务/定时备份**功能的用户（暂未实现）
- 需要**邮件通知/Webhook**告警的用户
- 需要**团队协作/权限管理**的企业用户
- 管理 **50+ 台服务器**的大规模运维（更适合 Ansible + Cockpit）
