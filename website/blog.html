LeePanel 系统兼容性说明：你的电脑能装吗？

大家好，这篇文章来聊聊 LeePanel 在各平台上的运行情况。毕竟下载之前，总得先知道自己能不能用。

Windows：10 起步，Server 也行

LeePanel 在 Windows 上最低需要 Windows 10，Windows 11 自然也没问题。

为什么不支持 Win7/8？因为 LeePanel 基于 Tauri v2 构建，界面渲染依赖 WebView2（微软的 Chromium 内核），而 WebView2 从设计上就放弃了 Windows 7/8/8.1。这不是我们不想支持，是微软不给支持。

如果你用的是 Windows Server，2019 和 2022 版本可以跑，但有两个前提：

  • 系统得是"桌面体验"模式（带 GUI 的那种，不是 Server Core 纯命令行）
  • 需要手动安装 WebView2 运行时

装好之后用起来和 Windows 10/11 桌面版完全一样，没有区别。

macOS：Intel 和 Apple Silicon 都支持

Mac 用户分两种情况：

  • Intel 芯片：最低 macOS 10.15 (Catalina)
  • Apple Silicon (M1/M2/M3/M4)：最低 macOS 11.0 (Big Sur)

Apple Silicon 版之所以起点更高，是因为这颗芯片本身就只从 Big Sur 开始才有。同样地，更老的 macOS（Mojave 10.14 及以下）也没办法支持，原因和 Windows 一样——框架底层的 WebKit 要求摆在那里。

Linux：需要桌面环境，不是纯命令行

Linux 版目前构建产物是 .deb 和 .AppImage，最低要求 glibc 2.35，对应的主流发行版大致是：

  • Ubuntu 22.04 及以上
  • Debian 12 (Bookworm) 及以上
  • Fedora 36 及以上
  • Arch Linux（滚动更新，永远最新）
  • CentOS/RHEL 9（勉强可以）

CentOS 8、Ubuntu 20.04、Debian 11 这些老一点的就不行了。

还有一点很重要：Linux 上必须有图形桌面环境（GNOME、KDE、XFCE 都行）。因为 WebKitGTK 渲染引擎需要 X11 或 Wayland 显示服务器，纯命令行的 headless 服务器是跑不了的。

不过话说回来，LeePanel 本身就是通过 SSH 远程管理服务器的工具。你被管理的那些 Linux 服务器不需要装 LeePanel，只要 SSH 能用就行。LeePanel 装在你自己的桌面电脑上就够了。

一张表总结

平台 最低版本 备注
Windows 10 (64-bit) 需 WebView2
Windows Server 2019 需桌面体验 + WebView2
macOS Intel 10.15 Catalina —
macOS Apple Silicon 11.0 Big Sur —
Linux glibc ≥ 2.35 需图形桌面环境

为什么不支持更老的系统？

一句话：WebView2 / WebKitGTK 不支持。

LeePanel 用的是 Tauri v2 框架，它不是一个独立的浏览器，而是调用操作系统自带的 Web 渲染引擎。这意味着它的兼容性上限直接取决于系统组件的支持范围。

想支持 Win7？得换成 Electron（内嵌完整 Chromium），但代价是重写整个应用，包体积从几十 MB 膨胀到 150MB+，内存占用也会翻倍。考虑到 Win7 已经停止支持 6 年了，这个投入产出比不太划算。

macOS 和 Linux 也是同理——框架层面的硬性限制，不是改个配置就能绕过的。

如果你在使用中遇到兼容性问题，欢迎到 GitHub Discussions 反馈。
