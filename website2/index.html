// capabilities

Everything a panel does. None of the risk.

SSH-first architecture

Every action is an SSH command fired from your machine. Sessions are independent — a slow server never blocks the others. Password & key auth, auto-reconnect, full xterm.js terminal included.

$ ssh root@203.0.113.7 → session #2 established

Real file manager

SFTP browsing with drag-and-drop upload, chunked large-file transfers, compress/extract, chmod, batch ops and favorites.

LNMP in one click

Nginx, MySQL/MariaDB, multi-version PHP-FPM. Install, start, stop, restart, reload — with live status dots.

Sites & SSL

Nginx vhosts, Let's Encrypt certificates, reverse proxy with WebSocket, rewrite rules, per-site PHP switching.

Databases & Redis

MySQL/MariaDB CRUD, user permissions, backup & restore. Redis key browsing across DB0–15 with TTL editing.

Docker, native

Container lifecycle, image pulls, log viewer and mirror-source config — no Portainer required.

Live monitoring

CPU, memory, swap, network, disk I/O, process list — streaming at your chosen interval.

Firewall & BBR

Manage ufw / firewalld / iptables rules visually, toggle BBR congestion control, reboot safely.

// why it's different

The panel that never touches your server.

01

Add your server

Host, port, password or key. Credentials stay in a local SQLite store on your machine — never on the box.

02

Connect over SSH

LeePanel opens its own session per server. No agent, no daemon, no exposed web port, no panel CVEs to patch.

03

Manage everything

Files, services, sites, databases, containers — rendered as a proper native UI instead of hand-typed commands.

✕ Traditional web panel

  • Installs a web server + panel code on the box
  • Opens ports 8888/8080 to the internet
  • Panel itself becomes the #1 attack surface
  • Root exploit = total compromise

LeePanel

  • Runs entirely on your desktop
  • Only your existing SSH port is used
  • Server stays exactly as you configured it
  • Uninstall = just close the app
